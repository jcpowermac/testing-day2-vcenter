# N-INF-01/02: Duplicate vCenter server values rejected

**File:** `test/e2e/infrastructure_validation_test.go`
**Labels:** `readonly`, `validation`, `p0`
**Component:** openshift/api

## Summary

Validates that the CRD's xValidation uniqueness rule rejects an Infrastructure spec containing two vCenter entries with the same `server` value.

## Actions

1. Skip if gate is disabled
2. Read current Infrastructure CR
3. Build a spec that duplicates the first vCenter entry (same server, different datacenter)
4. Submit dry-run patch and assert rejection with message containing `"vcenters must have unique server values"`

## Code

```go
It("should reject duplicate vCenter server values (N-INF-01/02)", func() {
    infra := currentInfrastructure()
    expectPatchRejected(duplicateFirstVCenterSpec(infra), "vcenters must have unique server values")
})
```

### Helper: `duplicateFirstVCenterSpec`

```go
func duplicateFirstVCenterSpec(infra *configv1.Infrastructure) *configv1.InfrastructureSpec {
    spec := vsphere.CloneInfrastructureSpec(infra.Spec)
    dup := vsphere.CloneVCenter(spec.PlatformSpec.VSphere.VCenters[0])
    dup.Datacenters = []string{"dup-dc"}
    spec.PlatformSpec.VSphere.VCenters = append(spec.PlatformSpec.VSphere.VCenters, dup)
    return &spec
}
```
